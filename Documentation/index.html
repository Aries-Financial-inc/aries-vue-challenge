Options Profit Calculator Documentation

Project Structure

  • App.vue: The main component that includes the CodingChallenge component.
  • components/CodingChallenge.vue: The component responsible for calculating and displaying the profit/loss data and the graph.
  • components/LineChart.vue: A wrapper around Chart.js to render the graph.

Implementation Details

App.vue

Purpose: Serves as the main entry point of the application.

Components Used: CodingChallenge

Data:

  • sampleData: Array of sample options data to be passed to the CodingChallenge component.

CodingChallenge.vue

Purpose: Calculates risk and reward for options strategies and displays the results.

Components Used: LineChart

Data:

  • chartData: Data used for rendering the chart.
  • chartOptions: Configuration options for the chart.
  • maxProfit: Maximum profit calculated from the options data.
  • maxLoss: Maximum loss calculated from the options data.
  • breakEvenPoints: Array of break-even points calculated from the options data.

Methods:

  • calculateRiskReward(): Method to calculate the risk and reward based on the options data.

Lifecycle Hooks:

  • mounted(): Calls calculateRiskReward() to initialize the chart data.

LineChart.vue

Purpose: Wrapper component for rendering a line chart using Chart.js.

Props:

  • chartData: Data to be displayed on the chart.
  • chartOptions: Configuration options for the chart.

Methods:

  • renderChart(): Initializes and renders the chart using Chart.js.

Watchers:

  • chartData: Re-renders the chart when the data changes.

Changes Made

  • Implemented the logic to calculate risk and reward for options strategies: This involved creating a method in CodingChallenge.vue that processes the input options data and computes the required values.
  • Integrated Chart.js to generate the risk and reward graph: The LineChart.vue component was created to wrap around Chart.js and handle the rendering of the graph.
  • Styled the application to display the maximum profit, maximum loss, and break-even points clearly: CSS styles were added to enhance the user interface and provide a better visual representation of the data.
  • Addressed the graph rendering issue: Ensured that the graph correctly displays upon loading the page without requiring any user interaction.
